# Provincial Assessor Tax Declaration History Retrieval System

Professional starter workspace for a Provincial Assessor system focused on retrieving complete property tax declaration history.

## Stack

- Frontend: Quasar, Vue 3, Vite, Pinia, Axios
- Backend: Laravel, PHP, Sanctum
- Database: PostgreSQL

## Project Structure

```text
provincial-assessor-system/
  frontend/   Quasar + Vue + Vite application
  backend/    Laravel API application
```

## Install Commands

Run these from this folder in your own terminal:

```powershell
cd frontend
npm install
```

```powershell
cd ../backend
composer install
```

If you want Laravel's full application skeleton generated by Composer instead of the starter manifest, run this before backend development:

```powershell
composer create-project laravel/laravel backend
cd backend
composer require laravel/sanctum spatie/laravel-permission
```

For the frontend full Quasar scaffold:

```powershell
npm create quasar@latest frontend
cd frontend
npm install axios pinia
```

## Recommended Database

Use PostgreSQL for this system. It fits long-term archival records, property history retrieval, reporting, and heavier search requirements better than a simple file or lightweight database.

Suggested database name:

```text
provincial_assessor_td_history
```

## MVP Included

- Property master records
- Tax declaration history per property
- Owner chain per declaration
- Document metadata per property/declaration
- Audit/activity log foundation
- Quasar records workspace with advanced search, record jacket, TD history, digitization queue, system activity, staff admin (admin), movement history, documents, audit export, print, and server dossier export
- Data entry for new property records
- Data entry for additional tax declarations on an existing property
- Supporting document registration and upload endpoint
- Property dossier API for complete record retrieval

## Retrieval Workflow

1. Search by TD number, ARP number, PIN, lot number, survey number, title number, owner, barangay, or municipality.
2. Select a property from the search results (match reason shown when applicable).
3. Review the complete property record jacket:
   - **TD History** — every tax declaration with linked documents, assessments, and data-entry events
   - **Digitization queue** — global workspace tab for all records awaiting scan
   - **Files** — documents grouped by tax declaration, with movement history per file
   - **Activity** — audit trail with staff name, TD filter, and CSV export

See `docs/ENHANCEMENT_PHASES.md` for the phased roadmap and environment variables.
4. Register existing physical records without a scan first (`For Scanning` status), then upload the scan from the Digitize tab.
5. Add tax declarations when the property is transferred, revised, subdivided, consolidated, or reclassified.
6. Print or export the complete property record.

## Local Run

The current local setup uses:

```text
Frontend: http://localhost:5173
Backend API: http://127.0.0.1:8002/api
Database: provincial_assessor_td_history
```

Backend:

```powershell
cd C:\provincial-assessor-system\backend
C:\php\php.exe artisan migrate --seed
C:\php\php.exe artisan serve --host=127.0.0.1 --port=8002
```

Frontend:

```powershell
cd C:\provincial-assessor-system\frontend
npm.cmd run dev -- --port 5173
```

The frontend reads `VITE_API_URL` from `frontend/.env`.
